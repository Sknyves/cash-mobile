Table de télémétrie non configurée

La table analytics_events n'existe pas encore dans votre base de données Supabase. Pour activer le tableau de bord de télémétrie, veuillez copier et exécuter le script SQL suivant dans l'éditeur SQL de votre console Supabase.

-- 1. Créer la table pour la télémétrie
CREATE TABLE IF NOT EXISTS analytics_events (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  device_id TEXT NOT NULL,
  event_name TEXT NOT NULL,
  properties JSONB DEFAULT '{}'::jsonb,
  created_at TIMESTAMP WITH TIME ZONE NOT NULL
);

-- 2. Activer Row Level Security (RLS)
ALTER TABLE analytics_events ENABLE ROW LEVEL SECURITY;

-- 3. Créer la politique pour autoriser l'insertion anonyme depuis l'application
CREATE POLICY "Allow anon insert" ON analytics_events
  FOR INSERT TO anon WITH CHECK (true);

-- 4. Créer la politique de lecture pour la clé d'administration
CREATE POLICY "Allow read for service_role" ON analytics_events
  FOR SELECT TO service_role USING (true);

Une fois le script SQL exécuté, cliquez sur "Actualiser" ci-dessus pour charger les données.
